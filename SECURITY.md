# Security Policy

## Supported Scope

This repository is an active product-building repository for **Lynknov**.  
Security reports are welcome for issues that may affect:

- authentication and authorization flows
- user data handling
- environment variable exposure
- database access control
- API endpoints and server actions
- deployment and infrastructure configuration
- dependency vulnerabilities that materially affect the project

## Reporting a Vulnerability

Please **do not open a public GitHub issue** for security vulnerabilities.

Instead, report security concerns privately to the repository owner / maintainer:

- **Primary contact:** Daffa Dhiyaulhaq Khadafi
- **Preferred channel:** private email or direct private contact
- **Subject format:** `[SECURITY][Lynknov] <short issue summary>`

When reporting, please include:

- a clear description of the issue
- affected files, routes, or components
- steps to reproduce
- impact assessment
- screenshots, logs, or proof of concept if relevant
- suggested mitigation, if available

## Response Expectations

Best effort targets for this repository:

- initial acknowledgement: within **3–7 days**
- triage and validation: as soon as reasonably possible
- remediation timeline: depends on severity, scope, and current sprint priorities

## Disclosure Policy

Please allow reasonable time for validation and remediation before any public disclosure.

The preferred process is:

1. private report
2. maintainer review
3. fix preparation
4. deployment or patch release
5. coordinated disclosure if needed

## Good Faith Research

Security research performed in good faith and reported responsibly is appreciated.

Please avoid:

- accessing data that does not belong to you
- disrupting services or degrading availability
- social engineering, phishing, or spam
- destructive testing against production systems

## Notes

This project may evolve quickly during active development.  
As a result, security posture, supported components, and infrastructure details may change over time.
