# Prompt — Sprint 0 Setup

You are helping me build Lynknov, an Interactive Business OS for founders, freelancers, creators, and business owners.

## Product framing
Lynknov is not a simple bio tool. It is a business identity platform built on 4 pillars:
1. Identity
2. Portfolio
3. Commerce
4. Lead Engine

## Current build goal
We are only building founder-first MVP v1.
Focus only on:
- auth
- onboarding
- profile
- page builder foundation
- portfolio showcase
- lead capture
- basic analytics
- publish flow

Do not add:
- marketplace
- template marketplace
- full CRM
- advanced AI
- complex commerce system
- team collaboration

## Your role
Act like a senior product-minded full-stack engineer.
Prioritize clean architecture, maintainability, and execution speed.
Do not overengineer.

## Tech assumptions
- Next.js App Router
- TypeScript
- Supabase
- Tailwind
- Server-first architecture where reasonable

## What I want from you now
1. Create the starter folder/file structure for the repo
2. Suggest exact package installation commands
3. Create the initial Supabase client setup
4. Create environment variable template
5. Create auth flow skeleton
6. Create onboarding route skeleton
7. Create dashboard route skeleton
8. Explain which files were created
9. Keep everything MVP-focused

## Important constraints
- Use domain-based structure under `features/`
- Keep shared UI in `components/`
- Keep utilities in `lib/`
- Keep code readable for an AI-assisted repo
- Avoid unnecessary abstractions

At the end, return:
- file tree
- created files
- code
- short implementation notes
