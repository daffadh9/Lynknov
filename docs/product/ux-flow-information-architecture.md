## Executive Summary

UX Lynknov tidak boleh terasa seperti builder generik. Experience harus terasa seperti user sedang merakit **mesin bisnis personal** yang ringkas dan strategis, bukan sekadar menyusun section di canvas.

Desain harus menjaga satu prinsip:
**setiap langkah harus membuat user lebih dekat ke halaman yang lebih kredibel dan lebih conversion-ready.**

## 1. Primary User Journeys

### Journey A — New creator onboarding
1. sign up
2. pilih use case
3. isi basic identity
4. pilih template / starter direction
5. tambahkan proof
6. tambahkan offer
7. aktifkan CTA / inquiry form
8. preview
9. publish

### Journey B — Visitor to inquiry
1. visitor masuk ke page
2. melihat positioning
3. melihat proof
4. memahami offer
5. klik CTA
6. isi inquiry form
7. masuk success state
8. lead masuk dashboard owner

### Journey C — Owner review leads
1. owner membuka dashboard
2. melihat lead list
3. membuka detail lead
4. melihat source / context
5. menindaklanjuti di luar sistem atau nanti via workflow lanjutan

## 2. Information Architecture

- Testimonials
- CTA / inquiry blocks
- optional basic links

## Authenticated Layer
- Onboarding
- Dashboard home
- Page builder
- Lead inbox
- Page settings
- Publish settings
- Account / billing

## 3. Dashboard Structure

### Recommended top-level nav
- Overview
- My Page
- Leads
- Analytics
- Settings

### Why this works
- tidak terlalu banyak
- mudah dipahami user non-teknis
- selaras dengan empat pilar

## 4. Onboarding Flow

### Goal onboarding
Membawa user ke published state secepat mungkin tanpa membuat mereka bingung.

### Step design
1. choose what you do
2. define positioning
3. choose a style direction
4. add proof
5. add offer
6. set CTA goal
7. publish first version

### Important UX note
Onboarding harus bertanya hal yang relevan dengan hasil. Jangan terlalu cepat melempar user ke builder kosong.

## 5. Builder Philosophy

Builder Lynknov harus lebih terarah daripada drag-and-drop bebas.

### Recommended interaction model
- section-based editor
- modular cards
- reorder sections
- edit content in structured panels
- real-time preview
- guided suggestions

### Why
- lebih cepat untuk target user
- lebih stabil secara desain
- memudahkan quality control
- memudahkan AI recommendations di masa depan

## 6. Suggested Page Anatomy

1. Hero / identity
2. Proof strip
3. About / what I do
4. Featured work / case studies
5. Testimonials
6. Offers / packages
7. FAQ or trust section
8. CTA / inquiry form
9. Footer identity

## 7. Publish Flow

### Publish states
- Draft
- Ready to publish
- Published
- Needs update

### Publish UX should include
- preview across device sizes
- page health checklist
- missing section warnings
- clear publish button
- shareable URL confirmation

## 8. Lead Capture Flow

### Recommended form structure
- name
- email
- project type
- budget / scope range
- timeline
- short brief
- CTA source context

### Good UX principle
Semakin high-intent user, semakin layak form sedikit lebih detail. Tapi jangan terlalu panjang untuk first conversion.

## 9. Analytics and Action Loop

### Minimal analytics early
- page views
- CTA clicks
- form opens
- form submissions
- offer clicks

### Why it matters
Analytics harus membantu user melihat apa yang bekerja, bukan sekadar angka cantik.

### Action loop
- user publish page
- Lynknov menangkap event
- Lynknov menunjukkan insight sederhana
- user memperbaiki headline / proof / CTA
- conversion meningkat

## 10. IA by Product Pillar

| Pillar | Core screens | UX goal |
|---|---|---|
| Identity | onboarding, profile, hero editor | membuat positioning jelas |
| Portfolio | project cards, testimonials, proof editor | membangun trust |
| Commerce | offer cards, CTA blocks, links | mendorong action |
| Lead Engine | form setup, leads list, analytics | menangkap dan memahami intent |

## 11. UX Risks

### Risk
Builder terlalu rumit.

### Mitigation
Gunakan structured editor, bukan full canvas complexity.

### Risk
Page terlihat cantik tapi tidak conversion-oriented.

### Mitigation
Selalu dorong user menambahkan proof, offer, dan CTA.

### Risk
Visitor bingung dengan banyak section.

### Mitigation
Beri opinionated recommended order dan checklist.

## 12. UX Success Criteria

UX dianggap berhasil jika:
- user bisa publish versi pertama dengan cepat
- visitor langsung memahami positioning user
- proof mudah ditemukan
- CTA jelas dan tidak tenggelam
- lead form terasa profesional
- dashboard owner mudah dipahami

## 13. Final UX Statement

**Experience Lynknov harus membuat user merasa sedang membangun business presence yang tajam dan siap menghasilkan peluang, bukan sekadar menyusun halaman.**
