# Lynknov Sprint 2–3 Execution Guide

## Kapan pakai Sprint 2
Pakai setelah Sprint 1 benar-benar stabil.

### Checklist sebelum Sprint 2
- auth berhasil
- onboarding berhasil
- profile row tersimpan
- draft page otomatis dibuat
- public page by slug sudah tampil
- lead form submit sudah masuk database
- analytics event dasar sudah masuk database

Kalau salah satu belum stabil, jangan masuk Sprint 2.

---

## Fokus Sprint 2
Sprint 2 bukan tentang tambah ide baru.
Sprint 2 adalah membuat builder dan dashboard benar-benar usable.

### Outcome Sprint 2
- founder bisa edit isi halaman dengan nyaman
- founder bisa kelola project portfolio
- founder paham status halaman
- founder bisa preview dan siap publish

---

## Kapan pakai Sprint 3
Pakai setelah Sprint 2 selesai dan flow edit halaman sudah nyaman.

## Fokus Sprint 3
Sprint 3 adalah penyempurnaan loop utama MVP.

### Outcome Sprint 3
- publish flow rapi
- public page terasa layak
- lead inbox dasar ada
- analytics ringkas berguna
- produk siap dipakai sendiri untuk testing

---

## Cara kerja terbaik dengan agent

### Untuk Sprint 2
- tempel prompt Sprint 2
- minta agent kerja bertahap per modul
- review setiap modul sebelum lanjut ke modul berikutnya

Urutan modul Sprint 2:
1. builder UX
2. section editing
3. page settings
4. portfolio management
5. dashboard usability

### Untuk Sprint 3
- tempel prompt Sprint 3
- minta agent fokus ke publish, public page, leads, analytics
- jangan izinkan agent melebar ke fase growth

Urutan modul Sprint 3:
1. publish flow
2. public page polish
3. lead inbox
4. analytics summary
5. QA hardening

---

## Cara review hasil agent
Setiap selesai sprint, cek 4 hal:
1. Apakah flow utamanya sekarang lebih utuh?
2. Apakah agent nambah fitur di luar scope?
3. Apakah struktur file masih rapi?
4. Apakah hasilnya benar-benar membantu founder use case?

---

## Setelah Sprint 3
Kalau Sprint 3 selesai, kamu masuk ke fase:
**Founder Internal Validation**

Yang kamu lakukan:
- pakai Lynknov untuk diri sendiri
- isi profil asli
- upload portfolio asli
- publish halaman asli
- coba capture lead
- ukur friction
- catat apa yang bikin flow terasa lambat/bingung

Baru setelah itu putuskan Sprint 4.
